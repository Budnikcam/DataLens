# Связка Google Таблица → дашборд

Цель: правка ячейки в **оригинальной** таблице клиента → после обновления страницы дашборд показывает новые данные.

**Таблица (источник истины):**  
[Family Garden / Sheet ID `1yzwXXxW1cVvc8t96a0uVXhz_KXR_thcd113y2U2tR9w`](https://docs.google.com/spreadsheets/d/1yzwXXxW1cVvc8t96a0uVXhz_KXR_thcd113y2U2tR9w/edit)

**Листы (только эти):**

| Лист | Содержимое |
|------|------------|
| `ТЕСТ 1` | Отчёт по проекту: общая информация, сводка, СМР, труд, комплектация, финансы, проблемы/риски |
| `ТЕСТ 2` | Задачи по блокам (ФИНАНСЫ/КАЗНАЧЕЙСТВО, ЗАКУПКИ/КАЦ, ПРАВОВЫЕ…) |

Apps Script: [`scripts/google_apps_script/Code.gs`](../scripts/google_apps_script/Code.gs) — **эвристический** разбор «как в отчёте», не плоские CSV-листы.

> Нормализованная книга `Dashboard_Mobile_Source` (`1dJtRHUOB2bAWj20FO_RIYYmono8syxEq3HRVy4PLKb4`) больше **не** используется для живой связки.

---

## Как это работает

1. **Google Apps Script** (привязан к этой таблице) читает `ТЕСТ 1` / `ТЕСТ 2` и отдаёт JSON в форме `data.json`.
2. **Публичный дашборд** (`index.html`) при наличии URL веб-приложения запрашивает этот JSON (fetch + JSONP fallback).
3. **Редактор** остаётся запасным: можно править и публиковать `data.json` через GitHub, если таблица недоступна.
4. Опционально **GitHub Action** раз в 15 минут подтягивает тот же JSON в `data.json` (бэкап / офлайн-файл).

Когда `sources.sheetsApiUrl` задан — для посетителей сайта **источник истины = Google Таблица**.

Подпись под шапкой: «Данные из Google Таблицы» / «Данные из файла».

## Хрупкие места парсера (важно для правок)

Парсер ищет **подписи строк и заголовки секций**. Менять формулировки «как удобнее» опасно — дашборд может потерять блок.

### ТЕСТ 1 — не переименовывать без правки скрипта

| Что ищем | Пример подписи |
|----------|----------------|
| Название объекта | `Наименование объекта` |
| Генподряд | `Генеральный подрядчик` |
| Договор / лимиты | `Стоимость работ … по договору`, `Стоимость по лимитам` |
| Даты | `Дата начала работ`, `Директивный срок завершения` |
| Сводка | секция `1.1. СВОДКА`, строки `Готовность`, `Плановая готовность`, `Численность рабочих: план/факт` |
| СМР | секция `… СТРОИТЕЛЬНО-МОНТАЖНЫЕ РАБОТЫ`, строка `Строительная готовность объекта…` |
| Труд | секция `… ТРУДОВЫЕ РЕСУРСЫ`, строки `Рабочие, чел.`, `ИТР, чел.` |
| Комплектация | секция `… КОМПЛЕКТАЦИЯ…`, категории `Инженерное оборудование` / `Технологическое оборудование` / `Материалы` / `Итого` |
| Финансы | секция `… ФИНАНСИРОВАНИЕ…`, метрики `Стоимость договора`, `Стоимость по лимитам`, `Оплачено подрядчику` |
| Риски | секция `… ПРОБЛЕМЫ, РИСКИ, РЕШЕНИЯ`, колонки `№ / Проблема / Влияние / Риск / Решение / Ответственный / Срок / Статус` |

Допустимо: менять **числа**, даты, тексты решений/проблем, статусы.  
Осторожно: проценты могут быть и как `54,71%`, и как доля `0,5471` — скрипт нормализует оба варианта. Кривые даты вроде `31.06.2027` / `24.01..2026` скрипт старается поправить (clamp / убрать лишние точки).

### ТЕСТ 2 — структура блоков

- Заголовок блока: номер + название (`1.0` + `ФИНАНСЫ/КАЗНАЧЕЙСТВО`) **или** строка `ПРАВОВЫЕ ВОПРОСЫ` + ответственный.
- Задачи — строки с маркером `-` (дефис) в тексте задачи.
- Колонки: `№ | Задача | Ответственный | Срок | Информация об исполнении`.
- Каденции в сроке: `еженедельно`, `еженедельно (по пятницам)`, `постоянно` — уходят в `deadline_type`, не в ISO-дату.
- Подблок `План на неделю:` склеивается с текущим блоком как `… / План на неделю`.

Если переставить колонки или убрать дефисы у задач — список на дашборде опустеет или поедет.

Локальная проверка разбора (если есть выгрузка xlsx):  
`python scripts/validate_test_sheets_parse.py`

## Настройка (Иван) — один раз

### 1. Доступ

- У Ивана: **Редактор** (или владелец) на таблицу выше.
- Маргарита правит **эту же** таблицу как обычно.

### 2. Установить Apps Script **в этой таблице**

1. Открыть [таблицу](https://docs.google.com/spreadsheets/d/1yzwXXxW1cVvc8t96a0uVXhz_KXR_thcd113y2U2tR9w/edit) → **Расширения → Apps Script**.
2. Вставить код из [`scripts/google_apps_script/Code.gs`](../scripts/google_apps_script/Code.gs) → Сохранить.
3. (Опционально) Выбрать функцию `debugBuild` → Выполнить — в журнале должен быть JSON с `project` и `tasks`.
4. **Развернуть → Новое развёртывание → Веб-приложение**:
   - Выполнять от имени: **Я**
   - У кого есть доступ: **Все** (Anyone)
5. Скопировать URL вида `https://script.google.com/macros/s/…/exec`.
6. Открыть URL в браузере — JSON с `project.name`, `tasks`, `risks`.

### 3. Вписать URL на сайт

1. Открыть [редактор](https://budnikcam.github.io/DataLens/edit.html).
2. Поле **«URL веб-приложения Google (Apps Script)»** → вставить `/exec` URL.
3. **«Сохранить на сайт»** (GitHub-токен, как раньше).
4. Через 1–2 минуты обновить дашборд — подпись «Данные из Google Таблицы».

Альтернативы: `config.js` → `sheetsApiUrl`, или query `?sheets=URL_ENCODED`.  
Порядок: `?sheets=` → `config.js` → `data.json.sources.sheetsApiUrl`.

### 4. (Опционально) GitHub Action

1. Repo → **Settings → Secrets → Actions** → `SHEETS_API_URL` = `/exec` URL.
2. Workflow: [`.github/workflows/sync-sheets.yml`](../.github/workflows/sync-sheets.yml).

## Ежедневная работа (Маргарита)

1. Правите ячейки в **ТЕСТ 1** / **ТЕСТ 2** (эта таблица).
2. На телефоне обновите дашборд (потянуть вниз / закрыть и открыть).
3. Данные подтягиваются при загрузке страницы.

---

## Запасной путь без таблицы

Редактор → правки → «Сохранить на сайт». Пока `sheetsApiUrl` пустой, дашборд читает `data.json`. Если URL задан, а Apps Script упал — откат на `data.json` / FALLBACK.
