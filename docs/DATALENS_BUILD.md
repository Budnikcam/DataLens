# Yandex DataLens — сборка мобильного дашборда

**Основной путь сдачи** (Looker Studio в РФ недоступен; топ-менеджеры на iPhone без VPN).

| | |
|---|---|
| Эталон UI | `preview/mobile-dashboard.html` |
| Стиль | `docs/STYLE.md` — только эти HEX |
| Данные | [Google Sheet](https://docs.google.com/spreadsheets/d/1dJtRHUOB2bAWj20FO_RIYYmono8syxEq3HRVy4PLKb4/edit) · `data/Dashboard_Mobile_Source.xlsx` · CSV в `data/` |
| Сервис | [datalens.yandex.ru](https://datalens.yandex.ru) или [datalens.yandex.cloud](https://datalens.yandex.cloud) |
| Документация | [yandex.cloud/ru/docs/datalens](https://yandex.cloud/ru/docs/datalens/) |

Цель: **2 вкладки/страницы**, ~390px, charts > text, карточка проекта + задачи.

---

## 1. Аккаунт и вход (5 минут)

1. Заведите / войдите в **Яндекс ID** (почта `@yandex.ru` или привязанный аккаунт).
2. Откройте один из входов:
   - https://datalens.yandex.ru  
   - https://datalens.yandex.cloud  
   - при редиректе в облако — https://datalens.yandex.cloud/ru/
3. При первом входе согласитесь создать экземпляр DataLens (бесплатный тариф Community / DataLens в организации — достаточно для этого заказа).
4. Создайте **воркбук** (рекомендуется новая модель «коллекции и воркбуки»), например: `Мобильный дашборд — ПРИРОДНЫЙ МИНИМАЛИЗМ`.

Официальный quickstart: [Начало работы](https://yandex.cloud/ru/docs/datalens/quickstart).

## 2. Подключение данных

Нужны листы / таблицы:

`project_card` · `kpi_summary` · `smr` · `labor` · `supply` · `finance` · `risks` · `tasks`

Локальные CSV: `data/01_project_card.csv` … `data/08_tasks.csv` (имена листов в xlsx — без префикса `01_`).

### Вариант A (предпочтительный): Google Sheets

Коннектор есть: [Создание подключения Google Sheets](https://yandex.cloud/ru/docs/datalens/operations/connection/create-google-sheets).

1. В воркбуке: **Создать → Подключение → Файлы и сервисы → Google Sheets**.
2. Доступ к таблице:
   - **Публичная по ссылке:** в Sheets → «Доступ» → «Все, у кого есть ссылка» (хотя бы «Читатель») → в DataLens **+ Добавить файл** → вставить URL из адресной строки браузера (не из окна «Поделиться», иначе откроется только первый лист).
   - **Закрытая таблица:** **Войти через Google** (OAuth) → разрешить доступ DataLens → вставить ссылку на файл.
3. В окне листов отметьте все нужные: `project_card`, `kpi_summary`, `smr`, `labor`, `supply`, `finance`, `risks`, `tasks` (пустое нельзя). Каждый лист = отдельная таблица в подключении.
4. **Заголовки столбцов** = Да.
5. Включите **Обновлять автоматически** (данные Sheets подтягиваются примерно раз в ≤30 мин; вручную — «Обновить данные»).
6. **Создать подключение**, имя например `GS Dashboard_Mobile_Source`.

Лимиты (актуально по доке): до 10 листов в работе, до 200 МБ на лист, до 300 колонок.

**Важно:** в Sheets задайте формат чисел (**Формат → Числа**), иначе DataLens может неверно угадать типы. Пустые ячейки в шапке/первых строках лучше заполнить (`0`, `none` и т.п.).

### Вариант B (надёжный fallback): CSV / Excel / Яндекс Диск

Если OAuth Google глючит, Sheets недоступен с аккаунта, или нужна гарантированная публикация без зависимости от Google:

1. **Создать → Подключение → Файлы** ([инструкция](https://yandex.cloud/ru/docs/datalens/operations/connection/create-file)).
2. **Загрузить файлы** — по одному CSV из `data/` **или** целиком `data/Dashboard_Mobile_Source.xlsx` (каждый лист Excel считается отдельным файлом; до 10 файлов на подключение).
3. Параметры CSV: кодировка `utf-8` / `utf-8-sig`, разделитель **запятая**, заголовки = Да.
4. Альтернатива: выгрузить те же файлы на **Яндекс Диск** и подключить через соответствующий файловый/облачный путь в DataLens (если доступен в вашем UI) — удобно для повторной загрузки с телефона.

Минус: обновление **ручное** (перезалить файл / заменить в подключении). Плюс: стабильно в РФ, без OAuth Google.

**Практичный гибрид:** сначала CSV/xlsx → первый чарт за 15 минут; параллельно подключить Sheets для «живых» данных.

## 3. Датасеты

Для каждого источника создайте датасет (**Создать → Датасет** → выбрать таблицу из подключения):

| Датасет | Источник | Назначение |
|---|---|---|
| `ds_project_card` | `project_card` | Название объекта, готовность, люди, сроки |
| `ds_kpi_summary` | `kpi_summary` | Сводные KPI / scorecards |
| `ds_smr` | `smr` | СМР / готовность по блокам |
| `ds_labor` | `labor` | План/факт людей |
| `ds_supply` | `supply` | Комплектация |
| `ds_finance` | `finance` | План/факт денег |
| `ds_risks` | `risks` | Риски (коротко) |
| `ds_tasks` | `tasks` | Статусы и список задач |

В датасете: проверьте типы (число / дата / строка), сохраните. Формулы — по минимуму; лучше считать в Sheets/Excel заранее.

## 5. Дашборд: 2 вкладки, phone-first (~390px)

**Создать → Дашборд.** Имя: `Мобильный — Проект / Задачи`.

### Вкладка 1 — «Проект»

Стек сверху вниз (одна колонка, ширина виджетов ~100% / узкий холст):

1. **Заголовок** — название объекта из `project_card` (коротко, без абзацев).
2. **Scorecards (indicator)** — 3–4 крупных числа: готовность факт %, дельта к плану, люди факт/план, ключевой KPI из `kpi_summary`. Цвет акцента `#2C5F4A`.
3. **Готовность** — bar / progress / gauge: факт `#2C5F4A`, план `#B8864E` (`project_card` / `smr`).
4. **Комплектация** — horizontal bar из `supply`.
5. **Финансы** — bar план/факт из `finance`.
6. **Риски** — компактный список или 2–3 indicator по статусу (`risks`); без длинных текстов.

### Вкладка 2 — «Задачи»

1. **Donut / pie** статусов задач (`tasks`) — выполнено `#6B8F7A`, в работе `#2C5F4A`, просрочено `#9E3B33`.
2. **Таблица или список** — имя задачи + срок (2–3 колонки максимум).
3. Опционально фильтр по статусу (селектор).

### Вёрстка под iPhone

- Виджеты **в один столбец**, отступы 16–24 px.
- Не ставить 2 колонки на узком экране.
- Крупные числа, подписи 1–3 слова.
- Проверка: Safari iPhone или Chrome DevTools → iPhone 12/13 (~390px), без горизонтального скролла.

Ориентир композиции: `preview/mobile-dashboard.html`.

## 6. Ссылка для просмотра на iPhone (Safari)

Два рабочих способа:

### A. Публичная ссылка (удобнее для топ-менеджеров без Яндекс-аккаунта)

1. Откройте дашборд → **Поделиться** / **Публичный доступ**.
2. Включите **Публичный доступ** / «Доступ по ссылке».
3. Скопируйте **публичную** ссылку (не URL из адресной строки редактора).
4. Отправьте Маргарите / руководителям. Открывается в Safari **без VPN и без логина**.

Дока: [Публикация объектов](https://yandex.cloud/ru/docs/datalens/concepts/datalens-public).  
Перед публикацией: нет лишней коммерческой тайны в чартах; публикация поддерживается для Files и Google Sheets (не для части спец-коннекторов вроде Billing/Metrica-as-source — к нам не относится).

### B. Доступ по приглашению (закрытый контур)

1. Пригласите пользователей в DataLens (Яндекс ID).
2. Выдайте права на воркбук/дашборд (просмотр).
3. Пришлите обычную ссылку на объект.

Для этого заказа предпочтителен **вариант A**.

Проверка: открыть публичную ссылку в режиме инкогнито / с телефона.

## 7. Как клиент обновляет данные

| Источник | Что делает клиент | Что видит дашборд |
|---|---|---|
| **Google Sheets** | Правит ячейки в [таблице](https://docs.google.com/spreadsheets/d/1dJtRHUOB2bAWj20FO_RIYYmono8syxEq3HRVy4PLKb4/edit) | Автообновление ≤30 мин или кнопка «Обновить данные» в подключении |
| **CSV / xlsx** | Присылает новый файл / правит Excel → вы перезаливаете в подключение Files | После замены файла и обновления датасета |
| **Структура** | Не переименовывать листы и заголовки колонок | Иначе сломаются поля чартов |

Кратко клиенту: «Меняете цифры в Google Таблице → через полчаса (или быстрее после ручного Refresh) дашборд на телефоне показывает новое. Ссылку не трогаем.»
