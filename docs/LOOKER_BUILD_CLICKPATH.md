## 0. Подготовка данных (один раз)

1. Откройте Google Таблицу с книгой `Dashboard_Mobile_Source` (или загрузите `data/Dashboard_Mobile_Source.xlsx` → Открыть в Google Таблицах).
2. Убедитесь, что есть листы с именами точно:
   - `project_card`
   - `smr`
   - `labor`
   - `supply`
   - `finance`
   - `risks`
   - `tasks`
3. В Looker Studio отчёте: **Ресурс → Управление добавленными источниками данных**.
4. Для каждого недостающего листа: **Добавить источник данных → Google Таблицы → та же книга → выбрать лист → Добавить**.
5. Имена полей оставьте как в заголовках CSV (ниже в каждом виджете).

## 1. Размер страницы и тема (сразу)

### 1.1 Холст под iPhone ~390px

1. Клик по **пустому фону** страницы (не по виджету).
2. Справа **Тема и макет** → вкладка **Макет**.
3. **Тип холста:** Фиксированный размер.
4. **Размер:** Пользовательский → **Ширина `400`**, **Высота `2100`** (страница «Проект»; для «Задачи» можно `400 × 1400` или одна длинная страница `400 × 3200` с якорями — см. §2).
5. **Ориентация:** Портрет.
6. **Поля:** все отступы **16**.

### 1.2 Цвета темы (ПРИРОДНЫЙ МИНИМАЛИЗМ)

1. Та же панель → вкладка **Тема**.
2. **Изменить текущую тему** (или Создать тему).
3. Задайте:

| Параметр темы | HEX |
|---|---|
| Primary / Акцент | `#2C5F4A` |
| Secondary | `#B8864E` |
| Фон страницы | `#F7F5F0` |
| Фон компонента | `#FFFFFF` |
| Граница | `#E8E0D8` |
| Текст основной | `#3A3A3A` |
| Текст вторичный | `#8A8580` |

4. Шрифты темы:
   - Заголовки: **Lora** (или Merriweather), 14–16
   - Текст / метрики: **Inter** (или Roboto)
5. **Сохранить**.

### 1.3 Фон страницы явно

1. Выделить страницу → **Стиль**.
2. Фон: `#F7F5F0`.
3. Сетка: выключить лишние линии; направляющие по желанию каждые 8 px.

## 2. Две «вкладки»: Проект / Задачи

Looker Studio **не делает нативные табы** как в HTML. Два рабочих варианта:

### Вариант A (рекомендуется) — 2 страницы + навигация

1. Внизу/слева списка страниц: **Добавить страницу**.
2. Переименовать:
   - страница 1 → `Проект`
   - страница 2 → `Задачи`
3. На **обеих** страницах: **Вставка → Элементы управления → Элемент навигации** (или кнопки-ссылки).
4. Две кнопки рядом (каждая ~ **180 × 48**):
   - «Проект» → Действие: Перейти на страницу `Проект`
   - «Задачи» → Действие: Перейти на страницу `Задачи`
5. Стиль кнопок:
   - Фон `#FFFFFF`, граница `#E8E0D8`, радиус 12
   - Текст активной: `#2C5F4A`, неактивной: `#8A8580`
   - Шрифт Inter 14 Medium
6. Скопировать пару кнопок на вторую страницу (Ctrl+C / Ctrl+V).

### Вариант B — одна длинная страница

1. Холст `400 × 3200`.
2. Сверху те же две кнопки с якорями на группы виджетов (если доступны закладки) — иначе Вариант A проще.

Дальше — **Вариант A**.

### 3.11 Список — Риски (`risks`)

Плотные таблицы на 5+ колонок **не использовать**. Компактный список:

## 4. Страница «Задачи»

### 4.1 Donut — статусы

1. **Вставка → Круговая диаграмма** → стиль **Кольцевая (Donut)**.
2. Источник: **`tasks`**.
3. Измерение: **`status`**.
4. Метрика: **Record Count** (или COUNT `task`).
5. Размер: **368 × 180** (кольцо слева ~120 + легенда).
6. Цвета сегментов вручную:
   - `В работе` → `#2C5F4A` (или градиент к `#6B8F7A`)
   - `Не исполнено` → `#9E3B33` (или `#9E3B33`→`#D4A574`)
7. Центральная подпись: Scorecard поверх с COUNT = **14** (или включите «Показывать метрику в центре», если есть).
8. Заголовок: `Статусы`.
9. Ожидаемо: **13** / **1**.

### 4.2 Фильтр статусов

1. **Вставка → Элементы управления → Раскрывающийся список** (или Чип-фильтр / Расширенный фильтр).
2. Источник: **`tasks`**, поле **`status`**.
3. Размер: три «кнопки» суммарно на ширину **368**, высота **44**.
4. Стиль контрола: фон `#FFFFFF`, border `#E8E0D8`, активный акцент `#2C5F4A`.

### 4.3 Список задач

1. **Вставка → Таблица**.
2. Источник: **`tasks`**.
3. Столбцы:
   - **`task`** (короткое имя)
   - **`deadline`** (или вычисляемое «дд.мм» / `deadline_type`)
4. Опционально скрытое поле для точки: **`status`** → цвет строки.
5. Размер: **368 × 520+** (прокрутка ok на телефоне).
6. Стиль как у рисков; высота строки ≥ **44**.
7. Сортировка: сначала `Не исполнено`, затем по `deadline`.
